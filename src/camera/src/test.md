## 测试一：key_center 静态跳变误差

### 目的

量化 `key_center`（相机坐标系下的目标体中心点）在不同高度和距离下的帧间跳变幅度，评估 RANSAC 平面拟合 + EMA 滤波的稳定性。

### 测试条件

| 条件 | 范围 |
|------|------|
| 目标高度（相机到目标面的 Z 方向距离） | 0.3 m ~ 0.8 m，每隔 0.1 m 取一个测试点 |
| 水平距离（目标在相机视野内的 XY 偏移） | 最近可测距离 ~ 最远可测距离，取近/中/远 3 档 |
| 每档采样帧数 | ≥ 200 帧 |
| 目标状态 | 静止（相机固定、目标 KFS 固定） |

### 测量指标

对每组（高度, 距离）的 N 帧 `key_center = (x, y, z)` 序列：

| 指标 | 公式 | 含义 |
|------|------|------|
| 均值 | $\bar{p} = \frac{1}{N}\sum p_i$ | 中心点平均位置 |
| 标准差 | $\sigma = \sqrt{\frac{1}{N}\sum \|p_i - \bar{p}\|^2}$ | 跳变幅度（越小越稳定） |
| 峰峰值 | $\max(p_i) - \min(p_i)$ | 最大跳变范围 |
| 跳变率 | 帧间 $\|\Delta p\| >$ 阈值的比例 | 突变帧占比 |

### 记录表格模板

| 高度 Z (m) | 水平偏移 | $\sigma_x$ (mm) | $\sigma_y$ (mm) | $\sigma_z$ (mm) | 峰峰值 (mm) | 跳变率 |
|------------|----------|-----------------|-----------------|-----------------|-------------|--------|
| 0.3        | 近       |                 |                 |                 |             |        |
| 0.3        | 中       |                 |                 |                 |             |        |
| 0.3        | 远       |                 |                 |                 |             |        |
| 0.4        | 近       |                 |                 |                 |             |        |
| ...        | ...      |                 |                 |                 |             |        |
| 0.8        | 远       |                 |                 |                 |             |        |

### 数据采集代码示例


## 测试二：坐标系变换精度验证

### 目的

验证通过多点标定计算 `XYZRPY` 偏置后，`set_bias_to_result()` 将相机坐标系下的点变换到目标坐标系的精度。

### 步骤

#### 2.1 固定相机和目标坐标系

- 相机固定在三脚架上，保持不动
- 在目标平面上标记目标坐标系原点 $O_t$ 和 X、Y 轴方向
- 目标坐标系朝向建议与相机坐标系一致（简化标定，仅需平移）

#### 2.2 多点标定（计算 XYZRPY 偏置）

在目标平面上选 **N ≥ 3 个不共线的标记点**（推荐 4~6 个），分别测量：

| 点编号 | 相机坐标系坐标 $(x_c, y_c, z_c)$ | 目标坐标系坐标 $(x_t, y_t, z_t)$ |
|--------|----------------------------------|----------------------------------|
| $P_1$  |                                  |                                  |
| $P_2$  |                                  |                                  |
| $P_3$  |                                  |                                  |
| ...    |                                  |                                  |

- **相机坐标**：将标记点放在相机视野中心附近，读取 `key_center` 多帧均值
- **目标坐标**：用物理量具（卷尺/卡尺）从目标坐标系原点测量

通过 Kabsch-Umeyama 算法（见 `method_math.h`）求解最优 $R, t$，填入 `params.yaml` 的 `bias_to_target`。

#### 2.3 移动目标 KFS，验证变换精度

标定完成后，将目标 KFS 移动到 **M ≥ 5 个不同位置**（覆盖不同高度和 XY 偏移），对每个位置：

| 步骤 | 操作 |
|------|------|
| ① | 相机采集 → `preprocess()` → `postprocess()` → 得到 `key_center`（相机系） |
| ② | 调用 `set_bias_to_result(params.bias_to_target)` → 得到 `out.x, out.y, out.z`（目标系） |
| ③ | 用物理量具在目标坐标系下实测该位置的坐标 $(x_{gt}, y_{gt}, z_{gt})$ |
| ④ | 计算误差：$e_x = \|out.x - x_{gt}\|$, $e_y = \|out.y - y_{gt}\|$, $e_z = \|out.z - z_{gt}\|$ |

### 记录表格模板

| 位置 | 变换后 X | 变换后 Y | 变换后 Z | 实测 X | 实测 Y | 实测 Z | $e_x$ | $e_y$ | $e_z$ | 总误差 $\sqrt{e_x^2+e_y^2+e_z^2}$ |
|------|----------|----------|----------|--------|--------|--------|-------|-------|-------|-------------------------------------|
| 1    |          |          |          |        |        |        |       |       |       |                                     |
| 2    |          |          |          |        |        |        |       |       |       |                                     |
| ...  |          |          |          |        |        |        |       |       |       |                                     |

### 评估标准

| 指标 | 合格线 |
|------|--------|
| X/Y 平均误差 | < 5 mm |
| Z 平均误差 | < 5 mm |
| 最大总误差 | < 10 mm |

## 附录：XYZRPY 标定算法（Kabsch-Umeyama）

已知 N 对点 $(p_i, q_i)$，其中 $p_i$ 为相机系坐标，$q_i$ 为目标系坐标：

$$\bar{p} = \frac{1}{N}\sum p_i,\quad \bar{q} = \frac{1}{N}\sum q_i$$

$$H = \sum_{i=1}^{N} (p_i - \bar{p})(q_i - \bar{q})^T$$

对 $H$ 做 SVD：$H = U S V^T$，则：

$$R = V U^T,\quad t = \bar{q} - R\bar{p}$$

从 $R$ 提取欧拉角得 `roll, pitch, yaw`，结合 $t$ 得完整 `XYZRPY`。
