# 计算机视觉中的多线程加速场景

## 判断标准：该任务是否满足以下条件？

| 条件 | 说明 |
|------|------|
| **CPU-bound（计算密集）** | 耗时在 CPU 运算而非 I/O 等待 |
| **可并行（数据独立）** | 各计算单元之间无依赖、无共享写入 |
| **颗粒度足够大** | 单任务耗时 >> 线程调度开销（通常 > 0.1ms） |
| **核数未饱和** | CPU 有空闲核心可调度 |

---

## ✅ 显著提速的场景

### 1. 多 ROI 独立处理（最典型）
```
YOLO 检测到 N 个目标
  ├─ ROI 1 点云提取 → RANSAC 拟合    }  串行: N×T
  ├─ ROI 2 点云提取 → RANSAC 拟合    }  并行: T（线程池）
  └─ ROI N 点云提取 → RANSAC 拟合    }
```
> **你的项目适用**：`set_pcl_clouds` 中每个 ROI 独立反投影，可并行。

### 2. 图像金字塔 / 多尺度推理
```
原图 → 缩放 0.5x → 检测（线程1）
     → 缩放 1.0x → 检测（线程2）
     → 缩放 2.0x → 检测（线程3）
```
> 适用于模板匹配、多尺度目标检测。

### 3. 前后端分离（生产者-消费者）
```
线程1: 相机采集 + 预处理     (I/O + 轻量CPU)
线程2: YOLO 推理              (GPU/CPU 密集型)
线程3: 点云处理 + 位姿解算    (CPU 密集型)
```
> **你的项目适用**：采集、推理、后处理可流水线化，吞吐量翻倍。

### 4. 批量图像处理
```
100 张图片
  ├─ 线程 1: 处理图 0..24
  ├─ 线程 2: 处理图 25..49
  ├─ 线程 3: 处理图 50..74
  └─ 线程 4: 处理图 75..99
```
> 离线数据集处理、bag 文件回放分析。

---

## ❌ 提速有限或无效的场景

| 场景 | 原因 |
|------|------|
| GPU 推理（YOLO/CNN） | GPU 本身已并行，CPU 多线程不能加速 GPU 计算 |
| 逐像素串行算法 | 像素间有依赖（如动态规划、图割）无法拆分 |
| 小任务频繁调度 | 任务 < 0.01ms 时线程切换开销 > 任务本身 |
| 内存带宽瓶颈 | 多线程同时读写大图像，带宽饱和后加线程无效 |
| 单核 CPU / 已满载 | 超线程反而因上下文切换变慢 |

## 你的项目可落地的并行点

```
                      ┌─ 采集线程（camera_read_depth）
主线程 ──→           ├─ YOLO 推理线程（detector.worker）
                      └─ 点云处理线程（set_pcl_clouds → postprocess）

当前: 串行，每帧 = T_capture + T_yolo + T_pcl
优化: 流水线，吞吐量 = max(T_capture, T_yolo, T_pcl)
```

### C++ 推荐方案

```cpp
// 轻量级：OpenMP（改动最小）
#pragma omp parallel for
for (const auto& roi : rois) {
    // 每个 ROI 独立反投影
}

// 中量级：std::async / std::thread（前后端分离）
auto future_yolo = std::async(std::launch::async, [&]{ return detector.worker(img); });
// 主线程做其他事情...
auto results = future_yolo.get();

// 重量级：线程池 + 任务队列（高吞吐离线处理）
// 推荐 TBB 或手写无锁队列
```

### 预期加速比

| 方案 | 场景 | 预期加速 |
|------|------|----------|
| ROI 并行（OpenMP） | 3~5 个 ROI | 2~3x |
| 前后端流水线 | 实时 30fps | 帧率上限 ↑50% |
| 两者叠加 | 多目标实时 | 3~4x |

## 注意事项

1. **OpenCV 内部已部分并行**：`cv::resize`、`cv::warpAffine` 等已用 TBB/IPP 多线程，外层再加线程可能竞争
2. **PCL 大部分单线程**：体素滤波、RANSAC 可放心并行
3. **ROS 回调是串行的**：`ros::spinOnce()` 需在主线程，数据可通过 `sensor_msgs` 共享指针传递
4. **调试难度**：多线程 bug（数据竞争、死锁）难以复现，建议先用 `std::async` 小步验证
