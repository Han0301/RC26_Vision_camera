## 1. 测试环境与前置条件

### 1.1 硬件与软件

| 项目 | 规格 |
|------|------|
| 相机 | RealSense D435i / 自选 |
| 分辨率 | 640×480 @ 30fps |
| 目标物 | 平整方块面（建议 15cm×15cm 以上），表面无明显反光 |


### 1.3 数据采集方式

- **实时相机**：`test1_fromframe()` 或 `test()`
- **bag 回放**：`test1_frombag()`，推荐用于同一场景复测
- 每帧记录：`x, y, z, bia_radian(°), timestamp, state`

### 1.4 评价指标

| 指标 | 公式 | 含义 |
|------|------|------|
| 均值 (avg) | $\bar{x} = \frac{1}{n}\sum x_i$ | 中心趋势 |
| 标准差 (std) | $\sigma = \sqrt{\frac{1}{n}\sum(x_i - \bar{x})^2}$ | 离散程度 |
| 峰峰值 (Δ) | $\max(x_i) - \min(x_i)$ | 最大跳变幅度 |
| 90% 分位范围 | $P_{95} - P_{5}$ | 剔除 10% 离群点后的范围 |
| 有效率 | $N_{valid} / N_{total}$ | preprocess+postprocess 均成功的帧比例 |

## 2. 测试一：点云深度边界有效性

### 2.1 问题背景

当前深度裁剪范围为 `[200, 2400] mm`。在边界附近存在两类问题：

```
相机 ─────────────────────→ 深度方向 (Z)
        │                    │
       200mm              2400mm
        ↑                    ↑
   最近裁剪线           最远裁剪线
```

| 边界问题 | 说明 |
|----------|------|
| **噪声闪入闪出** | 方块面在 ~2400mm 时，深度噪声 ±20mm 导致部分点云在 2380~2420 间跳动，每帧有效点数不稳定 |
| **远距离点云稀疏** | Z 越大，每像素对应的物理面积越大，点数密度 ∝ 1/Z²，RANSAC 拟合质量下降 |
| **近距离畸变** | Z < 300mm 时，相机透镜畸变最严重区域，`coeffs=0` 的假设失效 |
| **边界截断效应** | 方块面部分点在裁剪内、部分在外，拟合出的平面中心产生系统性偏移 |

```
远距离稀疏示意：

近距离 (0.5m):         远距离 (2.0m):
████████████           ██  ██  ██
████████████           ██  ██  ██
████████████           ██  ██  ██
点数多、密集            点数少、稀疏
RANSAC 稳定             RANSAC 易受噪点影响
```

### 2.2 测试步骤

| 步骤 | 操作 | 记录 |
|------|------|------|
| 1 | 方块面正对相机，法向量对齐光轴 | 初始角度 = 0° |
| 2 | 从 0.3m 开始，以 0.1m 步进至 2.5m | 每个距离记录 |
| 3 | 每个距离采集 ≥ 100 帧 | 使用 bag 回放或固定场景 |
| 4 | 记录每帧的有效点数、state、x/y/z | 见数据模板 |
| 5 | 观察 `state` 中各阶段失败原因 | 特别关注边界处 |

## 3. 测试二：不同距离下静态跳变

### 3.1 问题背景

即使方块面和相机完全静止，输出 $(x, y, z, \theta)$ 也会随时间抖动。来源：

```
传感器噪声 → 深度值抖动 → 点云位置浮动 → RANSAC 平面微动 → 中心/角度跳变
```

```
深度噪声链：
  CMOS 热噪 → 深度图 1~3mm RMS
  → 点云 Z 抖动（放大系数 ∝ Z²/fxb）
  → RANSAC 内点集合变化
  → 拟合平面法向量 ±0.1°~±0.5°
  → 平面中心 ±1~±5mm
```

### 3.2 测试步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 固定方块面，法向量正对相机 | 0° 倾斜 |
| 2 | 选择 3 个典型距离：0.5m, 1.0m, 1.5m, 2.0m | 覆盖近中远 |
| 3 | 每个距离采集 ≥ 500 帧（约 17 秒 @30fps） | 保证统计显著 |
| 4 | 保证光照恒定，相机已预热 ≥ 5 分钟 | 减少温漂 |
| 5 | 分别用当前参数和调优参数各测一轮 | A/B 对比 |

### 3.3 分析维度

#### 3.3.1 逐维度统计

对每个距离分别统计：

```
距离: 1.0m | 有效帧: 498/500
─────────────────────────────────────────────
指标          X(mm)      Y(mm)      Z(mm)     yaw(°)
avg           0.52      -1.38    1002.71      0.15
std           1.23       1.08       3.45      0.12
Δ(峰峰)       5.82       6.11      18.23      0.58
90%范围       3.12       3.45      10.56      0.31
```

#### 3.3.2 时序分析（关键）

绘制 500 帧的 X/Y/Z/yaw 时间序列，观察：

- 是否存在**低频漂移**（温漂导致）
- 是否存在**突发跳变**（某帧 RANSAC 收敛到错误平面）
- 跳变是否与 depth 图像中的某区域相关

```
理想（稳定）：          有问题（跳变）：
  ─────────────────      ───┐    ┌────────
                          └────┘
                          某帧突然跳变
```

#### 3.3.3 RANSAC 非确定性测量

在同一帧图像上重复调用 postprocess 10 次，记录结果范围：

```
重复10次 RANSAC（同一帧）:
  X: [0.48, 0.55]  Δ=0.07mm    ← 可忽略
  Y: [-1.41, -1.33]  Δ=0.08mm  ← 可忽略
  Z: [1001.2, 1003.8]  Δ=2.6mm ← 有影响
  yaw: [0.10°, 0.19°]  Δ=0.09° ← 有影响
```

### 3.4 减跳变策略

| 策略 | 方法 | 预期效果 |
|------|------|----------|
| **增加 RANSAC 迭代** | `MaxIterations: 1000 → 2000` | 减少拟合随机性 |
| **缩小体素** | `leaf_size_XY: 0.010 → 0.005` | 保留更多点，拟合更稳定 |
| **增大最小内点** | `MinPlaneInliers: 100 → 200` | 过滤不可靠平面 |
| **时域滤波** | 滑动窗口平均 (窗口=5帧) | 平滑高频抖动 |
| **异常帧剔除** | $\|z_i - z_{i-1}\| > 3\sigma$ 丢弃 | 过滤突发跳变 |

```cpp
// 时域滤波示例：滑动窗口
struct SlidingWindow {
    std::deque<double> window_;
    size_t max_size_;
public:
    explicit SlidingWindow(size_t n) : max_size_(n) {}
    double filter(double val) {
        window_.push_back(val);
        if (window_.size() > max_size_) window_.pop_front();
        return std::accumulate(window_.begin(), window_.end(), 0.0) / window_.size();
    }
};

// 使用
SlidingWindow filt_x(5), filt_y(5), filt_z(5), filt_ang(5);
out_filtered.x = filt_x.filter(out.x);
out_filtered.y = filt_y.filter(out.y);
```

## 4. 测试三：倾斜面稳定性

### 4.1 问题背景

方块面倾斜时，两种效应叠加：

```
正对 (0°):                倾斜 (30°):
┌─────────┐ 相机           ┌─────────┐
│█████████│ ←              │  █████  │ ↙ 相机
│█████████│                │ ███████ │
│█████████│                │█████████│
└─────────┘                └─────────┘

深度均匀                  深度渐变（近大远小）
点数均匀                  点数不均匀
RANSAC 稳定               RANSAC 可能偏向近端密集区域
```

| 倾斜效应 | 说明 |
|----------|------|
| **深度梯度** | 方块面近端和远端深度差可达 50mm+，单平面假设开始失效 |
| **点数不均** | 近端像素对应物理面积小、点数密；远端稀疏 |
| **RANSAC 偏移** | 随机采样偏向密集区域 → 拟合平面偏向近端的法向量 |
| **ROI 框变形** | YOLO 检测框在倾斜时可能不完整包含整个面 |
| **自遮挡** | 倾斜 > 45° 时面的一部分可能被自身遮挡 |

### 4.2 测试步骤

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 选择 3 个距离：0.8m, 1.2m, 1.8m | 代表近中远 |
| 2 | 每个距离下，从 0° 到 45°，以 5° 步进 | 8 个角度 |
| 3 | 每个 (距离, 角度) 组合采集 ≥ 300 帧 | 共 3×8×300 = 7200 帧 |
| 4 | 用支架+量角器精确控制角度 | 每次改变角度后等待 2s 稳定 |

#### 4.3.2 角度误差分析

`calc_deviation_angle` 计算的是拟合平面法向量与预设面法向量的夹角。在倾斜场景中，拟合出的角度应该与物理倾斜角度一致。

```
物理角度  测量角度(avg)  测量角度(std)  误差
0°        ?             ?              ?
5°        ?             ?              ?
10°       ?             ?              ?
...
45°       ?             ?              ?
```

**关注**：测量角度与物理角度的线性度 $R^2$，以及在哪个角度开始出现 > 2° 的系统偏差。

#### 4.3.3 中心点漂移

倾斜时，由于点数分布不均，拟合平面中心可能偏离几何中心。

```
测量方法：
1. 用物理方法标定方块面的几何中心 3D 位置 (ground truth)
2. 与 set_result 输出的 (x, y, z) 比较
3. 计算欧式距离偏差
```

### 4.4 大角度优化建议

```cpp

// 或者：对倾斜场景用加权 RANSAC（按点到相机距离加权）
// 让近端和远端点的权重均衡
```

### 5.3 采集脚本骨架

```cpp
void collect_stability_data(
    Ten::kfs_locator::Ten_set_result& plane_fiter,
    int num_frames,
    const std::string& output_csv)
{
    std::ofstream csv(output_csv);
    csv << "frame_id,timestamp_ms,x_mm,y_mm,z_mm,angle_deg,state,cloud_points\n";

    Ten::XYZRPY bias;  // 零 bias
    for (int i = 0; i < num_frames; ++i)
    {
        Ten::camera_frame frame = get_frame();  // 相机或 bag

        if (!plane_fiter.preprocess(frame))
            continue;

        if (!plane_fiter.postprocess())
            continue;

        auto out = plane_fiter.set_result(bias);
        auto clouds = plane_fiter.get_input_clouds();
        size_t total_pts = 0;
        for (auto& c : clouds) total_pts += c->size();

        csv << i << ","
            << get_timestamp_ms() << ","
            << out.x * 1000 << ","    // m→mm
            << out.y * 1000 << ","
            << out.z * 1000 << ","
            << out.bia_radian * 180.0 / M_PI << ","
            << plane_fiter.get_state() << ","
            << total_pts << "\n";

        ros::spinOnce();
    }
}
```

## 6. 判定标准与通过条件

### 6.1 测试一（边界有效性）

| 等级 | 条件 |
|:----:|------|
| ✅ 通过 | 0.5m~2.0m 有效帧率 ≥ 95%，且 Z std < 5mm |
| ⚠️ 警告 | 0.5m~2.0m 有效帧率 ≥ 85%，或某距离 Z std 5~10mm |
| ❌ 失败 | 某距离有效帧率 < 85%，或 Z std > 10mm |

**输出**：推荐的安全工作距离范围（如 0.5m~1.8m）

### 6.2 测试二（静态跳变）

| 等级 | X/Y std | Z std | yaw std |
|:----:|:-------:|:-----:|:-------:|
| ✅ 优秀 | < 1mm | < 3mm | < 0.15° |
| ⚠️ 可接受 | < 3mm | < 8mm | < 0.4° |
| ❌ 不可接受 | ≥ 3mm | ≥ 8mm | ≥ 0.4° |

### 6.3 测试三（倾斜）

| 等级 | 条件 |
|:----:|------|
| ✅ 通过 | 0°~30° 有效帧率 ≥ 90%，角度误差 < 2° |
| ⚠️ 警告 | 0°~30° 有效帧率 ≥ 80%，或角度误差 2°~4° |
| ❌ 失败 | > 30° 有效帧率 < 80%，或角度误差 > 4° |

## 附录：快速自检脚本

在正式测试前，先用以下脚本快速判断管线健康状态：

```cpp
void quick_health_check(Ten::kfs_locator::Ten_set_result& plane_fiter)
{
    Ten::XYZRPY bias;
    int ok = 0, fail_pre = 0, fail_post = 0;
    double sum_x = 0, sum_y = 0, sum_z = 0, sum_ang = 0;
    double min_x = 999, max_x = -999;  // 同理 y,z,ang

    constexpr int N = 200;
    for (int i = 0; i < N; ++i) {
        auto frame = get_frame();
        if (!plane_fiter.preprocess(frame)) { ++fail_pre; continue; }
        if (!plane_fiter.postprocess())     { ++fail_post; continue; }
        auto out = plane_fiter.set_result(bias);
        ++ok;
        sum_x += out.x; /* ... 更新 min/max ... */
    }

    std::cout << "有效: " << ok << "/" << N
              << " | pre失败: " << fail_pre
              << " | post失败: " << fail_post << std::endl;
    std::cout << "X Δ: " << (max_x - min_x)*1000 << "mm"
              << " | X avg: " << (sum_x/ok)*1000 << "mm" << std::endl;
    // ... 同理 Y, Z, angle
}
```

> 如果 `X Δ > 10mm` 或 `有效帧率 < 80%`，说明当前参数/距离组合不可信，需要先调优再进行详细测试。
