## 1. 概述

`ThreadPool::enqueue` 是一个**变参函数模板**，它接受任意可调用对象（函数、lambda、函数对象等）及其参数，将其提交到线程池中异步执行，并返回一个 `std::future<T>` 供调用者在未来获取执行结果。

```cpp
template<class F, class... Args>
auto enqueue(F&& f, Args&&... args) 
    -> std::future<std::invoke_result_t<F, Args...>>;
```

**核心链路**：调用者传入函数 + 参数 → `std::bind` 绑定 → `std::packaged_task` 包装（捕获返回值） → 包装为 `std::function<void()>` 塞入队列 → worker 线程取出执行 → 调用者通过 `std::future::get()` 获取结果。

## 2. 快速使用

### 2.1 最简示例

```cpp
#include "threadpool.h"

using namespace Ten::threadpool_test;

int main()
{
    ThreadPool pool(4, 100);  // 4 个线程，队列最大 100 个任务

    // 提交一个返回 int 的 lambda
    auto fut = pool.enqueue([](int a, int b) {
        return a + b;
    }, 3, 5);

    // 主线程继续做其他事...

    int result = fut.get();  // 阻塞等待，result = 8
    std::cout << "Result: " << result << std::endl;

    return 0;
}
```

### 2.2 提交普通函数

```cpp
double compute(double x, double y)
{
    return std::sqrt(x * x + y * y);
}

int main()
{
    ThreadPool pool(4, 100);

    auto fut = pool.enqueue(compute, 3.0, 4.0);
    double result = fut.get();  // result = 5.0

    return 0;
}
```

### 2.3 提交无返回值函数（返回 void）

```cpp
int main()
{
    ThreadPool pool(4, 100);

    auto fut = pool.enqueue([]() {
        std::cout << "hello from worker" << std::endl;
    });

    fut.get();  // 等待任务执行完毕（返回 void，但可以同步）

    return 0;
}
```

## 3. enqueue 源码逐行解析

```cpp
template<class F, class... Args>                              // ① 变参模板声明
inline auto enqueue(F&& f, Args&&... args)                    // ② 万能引用 + 尾置返回
    -> std::future<std::invoke_result_t<F, Args...>>          // ③ 推导返回值类型
{
    using ReturnType = std::invoke_result_t<F, Args...>;      // ④ 类型别名

    auto task = std::make_shared<std::packaged_task<ReturnType()>>(  // ⑤ 创建共享 packaged_task
        std::bind(std::forward<F>(f), std::forward<Args>(args)...)  // ⑥ bind 绑定参数
    );

    std::future<ReturnType> result = task->get_future();      // ⑦ 拿到 future（必须在入队前）

    {
        std::unique_lock<std::mutex> lock(queue_mutex_);
        condition_.wait(lock, [this] {                        // ⑧ 队列满则阻塞等待
            return tasks_queue_.size() < task_max_size_;
        });
        tasks_queue_.emplace([task]() { (*task)(); });       // ⑨ Lambda 包装后入队
    }

    condition_.notify_one();                                   // ⑩ 唤醒一个 worker 线程
    return result;                                             // ⑪ 返回 future 给调用者
}
```

| 步骤 | 操作 | 作用 |
|------|------|------|
| ① | `template<class F, class... Args>` | 声明模板参数：F 是可调用对象类型，Args 是参数包 |
| ② | `F&& f, Args&&... args` | 万能引用，保留参数的值类别（左值/右值） |
| ③ | `-> std::future<...>` | 尾置返回类型，利用 F 和 Args 推导返回值 |
| ④ | `using ReturnType = ...` | 提取纯返回类型，方便后续使用 |
| ⑤ | `std::make_shared<...>` | 在堆上构造 `packaged_task`，shared_ptr 共享所有权 |
| ⑥ | `std::bind(std::forward<F>(f), ...)` | 将函数与参数绑定为一个零参可调用对象 |
| ⑦ | `task->get_future()` | **必须在入队前**获取 future（之后 task 可能被执行完） |
| ⑧ | `condition_.wait(lock, ...)` | 背压：队列满时阻塞提交者 |
| ⑨ | `tasks_queue_.emplace(...)` | 将 packaged_task 包一层 `void()` lambda，入队 |
| ⑩ | `condition_.notify_one()` | 唤醒一个空闲 worker 线程去取任务 |
| ⑪ | `return result` | 返回 future 给调用者，后续可 `result.get()` 获取值 |

## 4. 核心语法详解

### 4.1 变参模板 (Variadic Template)

**语法**：`class... Args`、`Args&&... args`、`std::forward<Args>(args)...`

C++11 引入的变参模板允许模板接受任意数量的模板参数。

```cpp
// Args 是一个「参数包」(parameter pack)
template<class F, class... Args>
void enqueue(F&& f, Args&&... args);
```

- `class... Args` — 声明一个类型参数包
- `Args&&... args` — 函数参数包，每个参数都是万能引用
- `std::forward<Args>(args)...` — **包展开** (pack expansion)，相当于对每个参数分别调用 `std::forward`

```cpp
// 调用 enqueue(f, 1, 2.0, "hello")
// 展开为：enqueue(f, int&&, double&&, const char(&)[])
// forward 展开为：std::forward<int>(1), std::forward<double>(2.0), std::forward<const char(&)>("hello")
```

### 4.2 完美转发 `std::forward`

**用途**：将参数以其原始的值类别（左值/右值）转发给内部函数，避免不必要的拷贝。

```cpp
template<typename T>
void wrapper(T&& arg)
{
    // forward 保持 arg 的右值性：
    //   如果 arg 是右值引用 → forward 返回右值（移动语义）
    //   如果 arg 是左值引用 → forward 返回左值（拷贝语义）
    inner(std::forward<T>(arg));
}
```

**为什么需要它？** 如果不用 `forward`，所有参数会被当作左值传递，丢失移动优化的机会。

### 4.3 万能引用 `F&&`

**语法规则**：在模板参数 `T` 的推导上下文中，`T&&` 是**万能引用**（universal reference），不是普通的右值引用。

| 传入实参类型 | `T` 推导为 | `T&&` 实际类型 |
|-------------|-----------|---------------|
| 左值 `int x` | `T = int&` | `int& &&` → `int&`（引用折叠） |
| 右值 `42` | `T = int` | `int&&` |

```cpp
int x = 10;
enqueue(f, x);    // F&& 推导为 F&（左值引用），x 被拷贝绑定
enqueue(f, 42);   // F&& 推导为 F&&（右值引用），42 被移动绑定
```

> **引用折叠规则**：`& &` → `&`，`& &&` → `&`，`&& &` → `&`，`&& &&` → `&&`

### 4.4 `std::bind` 参数绑定

**头文件**：`<functional>`

**用途**：将函数与参数绑定在一起，返回一个新的零参可调用对象。

```cpp
// 原型
auto bound = std::bind(callable, arg1, arg2, ...);

// 示例
int add(int a, int b) { return a + b; }
auto bound_add = std::bind(add, 3, 5);
int result = bound_add();  // result = 8，等价于 add(3, 5)
```

在 `enqueue` 中：
```cpp
std::bind(std::forward<F>(f), std::forward<Args>(args)...)
//  将 f 与所有参数绑定 → 生成 ReturnType() 类型的零参可调用对象
//  这是 packaged_task<ReturnType()> 需要的形式
```

**占位符用法**（线程池中未使用，但值得了解）：
```cpp
using namespace std::placeholders;  // _1, _2, ...
auto f = std::bind(add, _1, 5);    // f(3) = add(3, 5) = 8
```

### 4.5 尾置返回类型 (Trailing Return Type)

**语法**：`auto func(...) -> ReturnType { ... }`

```cpp
// 传统写法：返回类型必须已知
int add(int a, int b) { return a + b; }

// 尾置返回：返回类型可以依赖参数类型
template<class F, class... Args>
auto enqueue(F&& f, Args&&... args) 
    -> std::future<std::invoke_result_t<F, Args...>>;
//   ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//   返回类型依赖于 F 和 Args 的推导结果
```

**为什么必须用尾置返回？** 因为 `F` 和 `Args` 只有在函数参数列表声明后才被推导，所以返回类型必须放在参数列表之后。

### 4.6 `std::invoke_result_t` 类型推导

**头文件**：`<type_traits>`（C++17）

**用途**：在编译期推导「用某类参数调用某可调用对象」的返回类型。

```cpp
// 语法
std::invoke_result_t<Callable, ArgTypes...>

// 示例
int add(int, int);
using R1 = std::invoke_result_t<decltype(add), int, int>;  // R1 = int

auto lambda = [](double x) { return x * 2; };
using R2 = std::invoke_result_t<decltype(lambda), double>; // R2 = double
```

在 `enqueue` 中：
```cpp
template<class F, class... Args>
auto enqueue(F&& f, Args&&... args)
    -> std::future<std::invoke_result_t<F, Args...>>;
//                  ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
//                  推导 f(args...) 的返回类型
```

> **C++14 兼容替代**：`std::result_of_t<F(Args...)>`（已废弃）

### 4.7 `std::packaged_task` 任务包装

**头文件**：`<future>`

**用途**：包装一个可调用对象，使其返回值可以通过关联的 `std::future` 获取。它是 `std::future` 和可调用对象之间的桥梁。

```cpp
// 创建 packaged_task：模板参数是函数签名（不是函数类型）
std::packaged_task<int(int,int)> task(add);

// 执行任务
task(3, 5);  // 执行 add(3,5)，结果存入内部共享状态

// 关联的 future 可以获取结果（必须在 task 执行前获取 future）
std::future<int> fut = task.get_future();
// ...
int result = fut.get();  // 阻塞直到结果就绪
```

**在线程池中的角色**：

```
┌─────────────────┐      ┌─────────────────────┐      ┌──────────────┐
│  调用者提交任务    │ ───→ │  packaged_task 包装   │ ───→ │ future.get() │
│  函数 + 参数      │      │  内部存储返回值        │      │  获取结果      │
└─────────────────┘      └─────────────────────┘      └──────────────┘
```

**为什么用 `std::shared_ptr<std::packaged_task<...>>`？**

- 任务执行完成后，packaged_task 的析构会使 future 变为 ready
- 用 `shared_ptr` 确保 task 的生命周期跨越入队到执行的全过程
- 在 `tasks_queue_` 中存储的是 `function<void()>`，它捕获了 `shared_ptr`，保证 task 不会被提前销毁

### 4.8 `std::future` 异步结果

**头文件**：`<future>`

**用途**：代表一个「未来会就绪」的异步操作结果。它是调用者和 worker 线程之间的通信通道。

```cpp
std::future<int> fut = pool.enqueue(compute, 3, 4);

// 方法 1：阻塞等待
int value = fut.get();  // 阻塞直到结果就绪，只能调用一次！

// 方法 2：轮询检查
while (fut.wait_for(std::chrono::seconds(0)) != std::future_status::ready) {
    // 做其他事...
}
int value = fut.get();

// 方法 3：超时等待
auto status = fut.wait_for(std::chrono::seconds(5));
if (status == std::future_status::ready) {
    int value = fut.get();
} else {
    // 超时处理
}

// 方法 4：等待（不取值）
fut.wait();   // 阻塞直到就绪
int value = fut.get();
```

| 方法 | 行为 | 适用场景 |
|------|------|---------|
| `get()` | 阻塞等待，获取值后 future 失效 | 必须拿到结果才能继续 |
| `wait()` | 阻塞等待（不获取值） | 需要同步但不关心返回值 |
| `wait_for(dur)` | 等待一段时间，返回状态 | 超时控制 |
| `wait_until(tp)` | 等待到时间点 | 截止时间控制 |
| `valid()` | 检查 future 是否有效 | 判断是否可调用 get() |

> **关键规则**：`future::get()` 只能调用一次，调用后 `valid()` 返回 `false`。如需多次获取，使用 `std::shared_future`。

### 4.9 `std::shared_ptr` 共享所有权

**为什么需要 `shared_ptr`？**

`packaged_task` **不可拷贝**（只可移动）。如果直接存入 `function<void()>`：
```cpp
std::packaged_task<int()> task(bound_func);   // 栈上对象
auto fut = task.get_future();
queue.push([t = std::move(task)]() { t(); }); // 移动进去
```
这可以工作，但 `make_shared` 更安全——它把对象创建在堆上，用引用计数管理生命周期，lambda 捕获 `shared_ptr` 即可。

### 4.10 Lambda 捕获

```cpp
tasks_queue_.emplace([task]() { (*task)(); });
//                    ^^^^         ^^^^^
//                  按值捕获      解引用指针，调用 operator()
```

- `[task]` — **按值捕获** `shared_ptr<package_task<...>>`，引用计数 +1
- `(*task)()` — 解引用 `shared_ptr` 得到 `packaged_task`，调用其 `operator()` 执行任务
- 任务执行完后 lambda 销毁 → `shared_ptr` 引用计数 -1 → 若为 0 则释放

## 5. 调用示例大全

### 5.1 Lambda 表达式

```cpp
ThreadPool pool(4, 100);

// 有返回值
auto f1 = pool.enqueue([](int x) { return x * x; }, 10);
std::cout << f1.get() << std::endl;  // 100

// 捕获外部变量
int base = 5;
auto f2 = pool.enqueue([base](int x) { return base + x; }, 10);
// f2.get() = 15

// 无返回值
auto f3 = pool.enqueue([]() { std::cout << "done\n"; });
f3.get();  // 等待完成
```

### 5.2 普通函数 & 静态函数

```cpp
int add(int a, int b) { return a + b; }

auto f = pool.enqueue(add, 3, 7);
int r = f.get();  // 10
```

### 5.3 成员函数

```cpp
class Calculator
{
public:
    int mul(int a, int b) const { return a * b; }
};

Calculator calc;
// 成员函数需要传递对象指针/引用作为第一个参数
auto f = pool.enqueue(&Calculator::mul, &calc, 6, 7);
int r = f.get();  // 42

// 或者用 lambda 包装（推荐，更清晰）
auto f2 = pool.enqueue([&calc](int a, int b) {
    return calc.mul(a, b);
}, 6, 7);
```

### 5.4 函数对象 (Functor)

```cpp
struct Multiplier
{
    int factor_;
    Multiplier(int f) : factor_(f) {}
    int operator()(int x) const { return x * factor_; }
};

Multiplier mul3(3);
auto f = pool.enqueue(mul3, 10);
int r = f.get();  // 30
```

### 5.5 并发提交多个任务，等待全部完成

```cpp
ThreadPool pool(4, 100);
std::vector<std::future<int>> futures;

for (int i = 0; i < 20; ++i) {
    futures.push_back(pool.enqueue([i]() {
        std::this_thread::sleep_for(std::chrono::milliseconds(100));
        return i * i;
    }));
}

// 逐个等待并收集结果
for (auto& fut : futures) {
    int result = fut.get();
    std::cout << result << " ";
}
```

### 5.6 任务间有依赖关系

```cpp
ThreadPool pool(4, 100);

// 任务 A
auto fa = pool.enqueue([]() { return 42; });

// 任务 B 依赖任务 A 的结果
auto fb = pool.enqueue([&fa]() {
    int a_result = fa.get();  // 阻塞等待 A 完成
    return a_result * 2;
});

int final = fb.get();  // 84
```

## 6. 常见问题与陷阱

### Q1: `future::get()` 只能调用一次

```cpp
auto fut = pool.enqueue([]() { return 42; });
int a = fut.get();  // OK
int b = fut.get();  // ❌ 抛异常 std::future_error
```

**解决方案**：使用 `std::shared_future` 多次获取。

```cpp
std::shared_future<int> shared_fut = pool.enqueue([]() { return 42; });
int a = shared_fut.get();  // OK
int b = shared_fut.get();  // OK
```

### Q2: 忘了调用 `get()` 会导致什么？

- **无返回值的任务**：任务仍然正常执行，不调用 `get()` 只是不等待完成
- **有返回值的任务**：任务仍然执行，但返回值被丢弃
- **析构时**：线程池析构会等待所有 worker 完成，已入队任务都会被处理

### Q3: 传入引用参数要注意什么？

```cpp
int value = 10;

// ❌ lambda 按值捕获了 value，但 value 可能在任务执行前被销毁
pool.enqueue([value]() { ... });

// ✅ 安全的做法：
// 方案1：用 shared_ptr 延长生命周期
auto sp = std::make_shared<int>(10);
pool.enqueue([sp]() { use(*sp); });

// 方案2：传值进去（小对象）
pool.enqueue([](int v) { use(v); }, value);
```

### Q4: 为什么 `get_future()` 必须在入队之前调用？

```cpp
// ✅ 正确顺序
std::future<ReturnType> result = task->get_future();  // 先拿 future
tasks_queue_.emplace([task]() { (*task)(); });        // 再入队

// ❌ 如果反过来，task 可能已被 worker 执行完毕，
// 此时再调用 get_future() 会抛异常（packaged_task 只能调用一次 get_future）
```

### Q5: 任务抛出异常怎么办？

```cpp
auto fut = pool.enqueue([]() -> int {
    throw std::runtime_error("oops");
    return 42;
});

try {
    int r = fut.get();  // 异常在此处重新抛出
} catch (const std::exception& e) {
    std::cerr << "Task failed: " << e.what() << std::endl;
}
```

`packaged_task` 会捕获任务中的异常，并在调用 `future::get()` 时重新抛出。

### Q6: Deadlock 风险

```cpp
ThreadPool pool(1, 10);  // ⚠️ 只有 1 个线程

// 任务 A 提交任务 B 并等待 B 完成
auto fa = pool.enqueue([&pool]() {
    auto fb = pool.enqueue([]() { return 42; });  // 任务 B 等待空闲线程
    return fb.get();  // 死锁！唯一的线程在等 B，B 没线程执行
});
fa.get();
```

**解决方案**：确保线程数足够，或将依赖关系改为异步回调。

## 7. 原理总结图

```
调用者线程                    线程池队列                   Worker线程
─────────                    ─────────                  ─────────

enqueue(f, args...)
    │
    ├─ bind(f, args...)
    │   生成 ReturnType() 零参函数
    │
    ├─ packaged_task<ReturnType()>
    │   包装零参函数，创建共享状态
    │
    ├─ task->get_future()
    │   拿到 future<ReturnType> ───────────────────────────────┐
    │                                                          │
    ├─ lambda [task](){(*task)();}                             │
    │   再包一层 void() lambda                                  │
    │                                                          │
    ├─ tasks_queue_.emplace(...)──→  [void()]                  │
    │                                                          │
    └─ return future ──────┐                                   │
                           │    ┌──────────────────────────┐    │
   future.get() 阻塞等待 ──┤    │  worker 取任务            │    │
                           │    │  执行 (*task)()           │    │
                           │    │  结果写入共享状态 ────────┼────┘
                           │    └──────────────────────────┘    │
                           │                                    │
   get() 返回结果 ←────────┴────────────────────────────────────┘
```
